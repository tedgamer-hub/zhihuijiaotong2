# IntelliDetect 后端技术文档

## 1. 项目概述

### 1.1 项目简介
IntelliDetect是一个基于Spring Boot开发的事故检测与管理系统，用于实现事故信息的采集、存储、查询和管理。系统提供了完整的事故信息管理功能，包括事故创建、查询、统计以及与IoT设备的通信能力。

### 1.2 技术栈

| 技术组件 | 版本 | 用途 |
|---------|------|------|
| Spring Boot | 2.6.13 | 应用框架 |
| Java | 1.8 | 开发语言 |
| Spring Data JPA | 2.6.13 | ORM框架 |
| MySQL | 8.0.33 | 数据库 |
| Lombok | 最新 | 代码简化工具 |
| Spring Security Crypto | 5.7.11 | 加密工具 |
| 华为云IoT设备接入服务 | 最新 | IoT设备通信 |

## 3. 核心功能模块

### 3.1 事故管理模块

#### 3.1.1 事故创建
- **功能**：接收事故信息并存储到数据库
- **流程**：
  1. 接收事故信息请求
  2. 设置事故描述时间为当前时间
  3. 保存事故信息到数据库
  4. 向IoT设备发送事故警告命令

#### 3.1.2 事故查询
- **功能**：提供事故信息的多维度查询
  - 分页查询所有事故
  - 查询单个事故详情
  - 查询最新事故信息
  - 查询事故统计数据

#### 3.1.3 事故统计
- **功能**：提供事故统计分析
  - 总事故数统计
  - 严重事故数统计
  - 今日新增事故数统计

### 3.2 IoT设备通信模块

#### 3.2.1 事故警告发送
- **功能**：当有新事故创建时，向IoT设备发送事故警告命令
- **实现**：调用华为云IoT设备接入服务的API发送命令

### 3.3 用户管理模块

#### 3.3.1 用户注册
- **功能**：接收用户注册信息并创建用户账号
- **流程**：
  1. 接收用户注册请求（用户名、密码、手机号、邮箱等）
  2. 验证注册信息的合法性和唯一性
  3. 加密存储用户密码
  4. 创建用户账号并返回用户信息

#### 3.3.2 用户登录
- **功能**：验证用户身份并返回用户信息
- **流程**：
  1. 接收用户登录请求（用户名、密码）
  2. 根据用户名查找用户
  3. 验证密码是否正确
  4. 返回用户详细信息

#### 3.3.3 用户信息查询
- **功能**：提供用户信息查询接口
  - 根据用户ID查询用户信息
  - 根据用户名查询用户信息

## 4. API接口说明

### 4.1 事故管理接口

#### 4.1.1 创建事故
```
POST /api/v1/accidents
请求体：
{
  "videoUrl": "视频URL",
  "imageUrl": "图片URL",
  "accidentDescription": "事故描述",
  "accidentDescriptionText": "事故描述文本",
  "accidentDescriptionState": "事故状态",
  "accidentX": 事故X坐标,
  "accidentY": 事故Y坐标
}

响应：
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": 事故ID
  }
}
```

#### 4.1.2 检查新事故
```
GET /api/v1/accidents/check-new

响应：
{
  "code": 200,
  "message": "有新事故产生",
  "data": 1  // 1表示有新事故，0表示无新事故
}
```

#### 4.1.3 获取事故列表
```
GET /api/v1/accidents?page=1&size=10000

响应：
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "total": 总记录数,
    "records": [事故列表],
    "page": 当前页码,
    "size": 每页记录数,
    "totalPages": 总页数
  }
}
```

#### 4.1.4 获取事故详情
```
GET /api/v1/accidents/{id}

响应：
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "id": 事故ID,
    "videoUrl": "视频URL",
    "imageUrl": "图片URL",
    "accidentDescription": "事故描述",
    "accidentDescriptionText": "事故描述文本",
    "accidentDescriptionTime": "事故时间",
    "accidentDescriptionState": "事故状态",
    "accidentX": 事故X坐标,
    "accidentY": 事故Y坐标

}
```

#### 4.1.5 获取事故统计

  "code": 200,
  "message": "查询成功",
  "data": {

    "severeAccidents": 严重事故数,

```

#### 4.1.6 获取最新事故

响应：
{
  "code": 200,
  "message": "查询成功",
  "data": [最新10条事故列表]

```

## 5. 数据库设计

### 5.1 事故表(accident_info)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 事故ID |
| video_url | VARCHAR(500) | NOT NULL | 视频URL |
| image_url | VARCHAR(500) | NOT NULL | 图片URL |
| accident_description | TEXT | NOT NULL | 事故描述 |
| accident_description_text | TEXT | NOT NULL | 事故描述文本 |
| accident_description_time | TEXT | NOT NULL | 事故时间 |
| accident_description_state | TEXT | NOT NULL | 事故状态 |
| accident_x | DOUBLE | | 事故X坐标 |
| accident_y | DOUBLE | | 事故Y坐标 |

### 5.2 用户表(user)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(100) | NOT NULL | 密码（加密存储） |
| email | VARCHAR(100) | NOT NULL, UNIQUE | 邮箱 |
| create_time | DATETIME | NOT NULL | 创建时间 |
| update_time | DATETIME | NOT NULL | 更新时间 |

## 8. 开发说明

### 8.1 编码规范
1. 遵循Java编码规范，使用驼峰命名法
2. 使用Lombok简化代码，减少样板代码
3. 接口返回统一格式，包含code、message、data字段
4. 异常处理统一，提供友好的错误信息

### 8.2 安全注意事项
1. 密码使用Spring Security Crypto进行加密存储
2. API接口添加跨域配置
3. 敏感信息（如数据库密码、IoT密钥）通过配置文件管理
